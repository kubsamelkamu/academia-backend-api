# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it privately so we can investigate and address it before a public disclosure.

- **Email:** contact@kubsamelkamu.dev
- **What to include:**
  - A clear description of the issue and potential impact
  - Steps to reproduce (proof-of-concept if possible)
  - Affected endpoints/modules and any relevant logs
  - Your environment details (Node version, OS, deployment options)

We aim to acknowledge reports within **72 hours** and provide an initial assessment within **7 days**.

## Supported Versions

Security fixes are applied to the latest `main` branch. If you are running an older revision, please update to the latest commit.

## Coordinated Disclosure

Please avoid posting the vulnerability publicly (GitHub issues, social media, etc.) until we have had a chance to release a fix.

Thank you for helping keep this project and its users secure.

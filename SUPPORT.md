# Support

Thanks for using Academic Project Management & Collaboration Platform.

## Where to get help

- **Questions / usage help:** Open a GitHub Issue with the `question` label.
- **Bug reports:** Open a GitHub Issue with clear reproduction steps.
- **Security issues:** Do **not** open an issue. Please email contact@kubsamelkamu.dev and see `SECURITY.md`.

## What to include in bug reports

- Expected vs actual behavior
- Steps to reproduce
- Logs and stack traces (redact secrets)
- Environment info (Node version, OS, DB/Redis versions)

## Response expectations

This is an open-source project maintained on a best-effort basis. Critical security reports are prioritized.
